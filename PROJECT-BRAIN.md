## 1. What this is
Biochemical laboratory management backend. NestJS REST API for multi-tenant lab operation (patient CRUD, doctor/insurer management, practice catalog, order + result tracking, PDF report generation). Supports multiple laboratories, user role-based access (admin, recepcion, bioquimico, super), and cost management via UB (Unidad Básica) pricing. Multi-tenant via labId (integer, nullable for super role). Soft-delete + RLS enforced at database level.

## 4. Conventions (how code is actually written here)
- **Imports:** Use `@/` path alias (e.g., `@/auth/session`, `@/db/schema`); configured in tsconfig.json
- **Services:** All injectable via `@Injectable()`; inject `@Inject(DATABASE) private readonly db: Db` in constructor to access Drizzle query builder
- **Controllers:** Use NestJS decorators (`@Get()`, `@Post()`, `@Patch()`, `@Delete()`); apply `@ApiBearerAuth()` + `@ApiTags()` for Swagger docs; extract `@CurrentUser()` for Session
- **Validation:** class-validator decorators on DTOs (class-based, auto-transformed by ValidationPipe); Zod for env schema only (AppConfig at startup)
- **Error handling:** Throw NestJS exceptions (NotFoundException, ConflictException, UnauthorizedException); framework auto-maps to HTTP status codes
- **Authentication:** Extract `@CurrentUser()` Session; check role via `@Roles('admin')` decorator or via TenantService.resolve(); use `requireLabId(session)` to throw 403 if labId is null (super-user guard)
- **Multi-tenancy:** labId is tenant key (nullable integer); super role has labId=null. All queries filter by `and(eq(table.labId, labId), isNull(table.deletedAt))`; no super exceptions unless explicitly designed
- **Soft-delete:** Never hard-delete; set `deletedAt` timestamp. Query filter: `isNull(table.deletedAt)`
- **Database queries:** Drizzle query builder only (`select().from(table).where(...).orderBy(...).limit(...)`) with `and()`, `or()`, `eq()`, `ilike()`, `desc()`, `isNull()` utilities
- **Naming:** 
  - DB tables & columns: snake_case (patient, order_practice, nbu_code)
  - TypeScript types/interfaces: PascalCase (Patient, OrderPractice)
  - DTOs: `Create*Dto`, `Update*Dto`, `*Dto`
  - Services: `*.service.ts`, classes as `PascalCase + Service`
  - Routes: kebab-case (`/api/users`, `/api/patients/{id}`, `/api/orders`)
- **Timestamps:** Always include createdAt, updatedAt, deletedAt (nullable); timezone is Argentina (UTC-3) — anchored in src/tz.ts (must load first in main.ts)
- **Money:** Use MoneyString (string type with 2 decimals, no symbol); Decimal.js for arithmetic; never raw floats for prices
- **Tests:** Jest + ts-jest; `*.spec.ts` files; focus on domain logic (money calculations, status transitions, auth policies) + guard behavior

## 7. Decisions & why
- **Drizzle ORM (not TypeORM):** Lightweight, type-safe SQL builder; schema-as-code with auto-generated migrations; better fit for Supabase
- **Supabase Auth (not Passport):** Pre-built JWT verification, user management, multi-device sessions; JwtVerifier in-memory caching (60s TTL) reduces Supabase API round-trips
- **Multi-tenant via labId:** Tenant key is nullable integer (null = super role). Hard filters on all queries `eq(table.labId, labId)`. Super users (role='super', labId=null) are cross-tenant; `requireLabId(session)` throws 403 to block super users from lab-specific endpoints
- **Soft-delete + RLS:** Business requirement for audit trail. `isNull(deletedAt)` filters all queries. RLS policies enabled on DB (PostgreSQL SECURITY level = ROW), but enforcement is code-level via labId filters (backend bypasses RLS with service_role key)
- **Zod for config validation:** Fail-fast at startup if env vars invalid; single source of truth for environment schema
- **Helmet + Compression:** Security headers + bandwidth optimization; applied globally in main.ts
- **Biome (not ESLint/Prettier):** Single unified tool for lint + format; faster, simpler config
- **Decimal.js for money:** Arbitrary-precision arithmetic; avoids JS float precision issues in pricing/billing calculations
- **@react-pdf/renderer:** Server-side PDF generation (no headless browser); generates lab reports synchronously
- **pnpm workspace-friendly:** Faster lock files; monorepo-ready if needed later
- **Migration 0005 captured drift:** Previous `db:push` commands left practice table with orphaned columns (parent_id, reference_value, methodology, is_elaborated) not in Drizzle schema. Migration 0005 explicitly captured this state, then schema was updated to include them

## 10. Gotchas & landmines
- **TZ anchor must be first in main.ts:** `src/tz.ts` runs BEFORE any date parsing. It sets `process.env.TZ = 'America/Argentina/Buenos_Aires'`. All date operations inherit this. If tz.ts runs late, timezone offsets will be wrong
- **DATABASE token is global:** Database module is global; client is singleton with no per-request isolation. Row-level access is enforced by code-level labId filters and (nominally) by RLS at DB level
- **Drizzle migrations are destructive in dev:** `pnpm db:reset` drops both public AND drizzle schemas. Migration history is wiped; next `db:migrate` re-applies all migrations from 0000. Never run in production
- **db:push without migration = drift:** Previous `db:push` commands synced schema but created NO migration file. This caused migration 0005 to capture orphaned columns (parent_id, reference_value, methodology, is_elaborated). Always use `db:generate` + `db:migrate` in production
- **Supabase JWT cache is 60s TTL:** JwtVerifier holds in-memory cache of (token → session). User role changes have ~1min delay. Cache is per-process (not shared across instances)
- **RLS is enabled but policies are empty:** PostgreSQL SECURITY level = ROW, but no CREATE POLICY statements. Backend connection uses service_role key (bypasses RLS). Frontend would get 403 on anon key (correct behavior)
- **Super user labId is NULL, not 0:** `requireLabId(session)` throws 403 if `session.labId === null`. This blocks super users from lab-specific endpoints
- **Order practice immutability enforced by DB trigger:** UPDATE on order_practice snapshots raises exception (`trg_order_practice_immutable`). To change a line, DELETE + re-INSERT
- **class-validator must be precise:** DTO decorators must match actual types; mismatch causes runtime transform failures in ValidationPipe
- **pnpm lock file is large:** pnpm-lock.yaml is monorepo-friendly but huge; review diffs carefully if dependencies change
- **Supabase pooler port 6543:** DATABASE_URL connects to pgBouncer (port 6543), not raw Postgres (5432). Configured in Supabase project settings
- **Soft-delete scope varies:** Some queries use service helpers, others explicitly `isNull(deletedAt)`. Be consistent when adding queries
